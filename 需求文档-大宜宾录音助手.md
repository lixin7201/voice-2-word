日期：2026-06-15

## 1. 项目目标

大宜宾录音助手是一个公司内部使用的 Chrome 扩展。第一阶段只做 Chrome 扩展，不做 ztools 插件，不做独立移动端。

它要解决的问题是：员工在 Plaud 网页端或任意网页端播放录音后，插件可以捕获音频，上传到公司局域网后端，自动完成录音转文字、会议纪要、AI 总结、红娘客户画像、招聘客户跟进记录生成，并把结果长期保存在员工自己的历史记录中。

目标体验接近 Plaud：

1. 用户打开录音网页。
2. 点开 Chrome 侧边栏插件。
3. 点击“监听当前页录音”。
4. 用户播放网页上的录音。
5. 插件捕获候选录音。
6. 如有多个候选，用户选择一个。
7. 插件上传录音并开始转写。
8. 用户可以收起插件，不需要等待。
9. 后端继续完成转写、总结、结构化跟单。
10. 用户之后再打开插件，可以看到历史结果、复制、备注、导出。

## 2. 第一阶段范围

### 2.1 必须做

1. Chrome 扩展侧边栏 UI。
2. 员工工号登录。
3. 员工密码默认 `dayibin`，允许用户后续自行修改，但不强制首次修改。
4. 管理员可以新增、删除、禁用、恢复员工，并选择员工所属部门。
5. 管理员可以调整员工角色。
6. 部门领导按部门固定配置。
7. 普通员工只能看自己的录音。
8. 部门领导可以看自己负责部门的录音。
9. 离心和练团长拥有最高权限，可以看全部部门和全部员工。
10. 支持监听任意网页端录音。
11. 支持 Plaud 网页端录音，前提是用户自己已经登录 Plaud 网页端。
12. 支持手动上传录音文件。
13. 支持格式：`mp3`、`m4a`、`wav`、`aac`、`flac`、`ogg`、`opus`，视频含音频的 `mp4`、`mov`、`webm` 可以作为加分项。
14. 录音上传后转成可供阿里云 DashScope 识别的临时音频链接。
15. 复用 ai-dayibin 的录音总结能力和 UI 思路。
16. ASR 使用阿里云 DashScope Fun-ASR。
17. LLM 总结复用 ai-dayibin 模型适配层，EasyAI/反代为主，Kimi 为兜底。
18. 支持会议纪要、业务复盘、通用客户跟进、红娘客户画像、招聘跟进记录。
19. 录音原文、AI 总结、结构化跟单、备注都要保存。
20. 支持下载导出。
21. 支持长期保存，第一阶段不做自动清理。

### 2.2 第一阶段不做

1. 不做 ztools 每晚 9 点自动复盘的完整功能，只预留接口。
2. 不自动写入云客、CRM、飞书表格或其他业务系统。
3. 不做复杂计费系统。
4. 不做移动端。
5. 不做浏览器外本地录音监听。
6. 不承诺支持 DRM 加密音频。

## 3. 部署形态

### 3.1 局域网服务

后端服务运行在离心的 Mac 或办公室局域网内可访问的机器上，类似当前 ai-dayibin 的局域网模式。

当前本机已占用端口包括 `8080`、`8082`、`8083`、`8090`、`5000`。本项目建议使用：

```text
http://lixindemac-studio.local:8127
```

端口：`8127`

如果开发时发现 `8127` 被占用，则顺延使用 `8128`，但文档和配置应统一更新。

### 3.2 系统组成

1. Chrome 扩展：只负责用户交互、网页录音捕获、上传、历史展示、导出下载触发。
2. 后端 API：负责登录鉴权、任务管理、文件上传、R2 存储、ASR、LLM 总结、权限校验、导出。
3. 本地 Supabase：作为局域网内业务数据库，存员工、部门、角色、录音记录、转写、总结、备注、权限关系。
4. Cloudflare R2：存录音原文件、导出文件、转写中间文件。
5. ai-dayibin 复用模块：复用录音总结模板、模型适配层、导出逻辑和 UI 交互经验。

### 3.3 密钥要求

插件源码中不得出现以下密钥：

1. 阿里云 DashScope API Key。
2. EasyAI API Key。
3. Kimi API Key。
4. Cloudflare R2 Access Key。
5. Supabase service role key。

这些密钥只允许存在于后端 `.env` 中。Chrome 扩展只拿后端颁发的登录 token。

1. 打开插件管理后台。
2. 进入“员工管理”。
3. 点击“新增员工”。
4. 输入花名。
5. 选择部门。
6. 选择角色，默认普通员工。
7. 初始密码自动设为 `dayibin`。
8. 保存。
9. 系统生成员工登录账号。

管理员删除员工动作：

1. 打开“员工管理”。
2. 搜索员工花名。
3. 点击“停用员工”。

1. 用户点击右上角头像或花名。
2. 选择“修改密码”。

6. 保存。
7. 后端校验旧密码。

### 6.3 Token

后端登录成功后返回访问 token。

插件保存：

1. `accessToken`
2. `refreshToken` 或后端 session id
3. 当前用户信息
4. 后端地址

保存位置使用 Chrome storage，不写入页面 localStorage。

## 7. Chrome 扩展 UI

### 7.1 设计风格

用户明确要求：每个人都有插件录音 UI 界面，界面美观大方，使用“液态玻璃质感”。

设计关键词：

1. 液态玻璃。
2. 清透。
3. 大宜宾内部工具。
4. 像 Plaud 一样专业。
5. 不要廉价后台感。
6. 不要花哨营销页。
7. 信息密度适中。
8. 操作按钮清晰。

视觉建议：

1. 侧边栏宽度适配 Chrome Side Panel，默认 380 到 420 px。
2. 背景使用浅色半透明玻璃层，不做厚重纯色。
3. 卡片使用 `backdrop-filter: blur(20px)`，边框使用半透明白色或浅灰。
4. 主按钮使用低饱和蓝绿或蓝紫渐变，但不要全页面单一紫色。
5. 录音状态使用动态波纹或环形进度。
6. 历史列表使用紧凑卡片。
7. 详情页用 Tab 分层：总结、逐字稿、跟单、备注、导出。
8. 重要状态要有清楚的颜色区分：处理中、完成、失败、待选择。

## 8. 录音捕获功能

### 8.1 捕获目标

6. 网络请求中的音频文件。
7. 手动上传的本地录音文件。

### 8.2 捕获动作

用户动作：

1. 打开目标网页。
2. 打开插件。
3. 点击“监听当前页录音”。
4. 插件显示“监听中”。
5. 用户点击网页上的播放按钮。
6. 插件收集当前 Tab 的媒体资源。
7. 如发现多个候选，用户选择要识别的录音。
8. 点击“开始识别”。

### 8.3 捕获技术要求

Manifest 权限建议：

1. `sidePanel`
2. `storage`
3. `activeTab`
4. `scripting`
5. `tabs`
6. `downloads`
7. `webRequest`

Host permissions：

1. `http://*/*`
2. `https://*/*`
3. `http://lixindemac-studio.local:8127/*`

候选识别规则：

1. URL 后缀匹配音频或视频扩展名。
2. `Content-Type` 包含 `audio/`。
3. `Content-Type` 包含 `video/` 且文件用于播放。
4. `audio.currentSrc` 或 `video.currentSrc`。
5. `source.src`。
6. `performance.getEntriesByType("resource")` 中的媒体资源。

候选去重规则：

1. URL 完全相同只保留一次。
2. 去掉 query 后相同但大小不同，保留大小更大的。
3. 小于 50 KB 的候选默认折叠到“疑似无效资源”。

## 15. 权限规则

### 15.1 录音查看权限

普通员工：

```text
只能查看 owner_employee_id = 当前员工 id 的记录。
```

部门领导：

```text
可以查看 owner_department_id 属于自己 lead 部门的记录。
```

管理员和老板：

```text
可以查看全部记录。
```

### 15.2 员工管理权限

只有 `admin` 和 `boss` 可以进入员工管理。

### 15.3 导出权限

能查看的记录才能导出。

导出他人记录必须写入审计日志。

### 15.4 备注权限

1. 记录所有者可以新增和编辑自己的备注。
2. 部门领导可以给部门记录追加领导备注。
3. 管理员和老板可以追加管理备注。
4. 不同人的备注要保留作者，不互相覆盖。

## 16. 导出要求

### 16.1 导出类型

1. 逐字稿。
2. 总结。
3. 跟单。
4. 完整记录。

### 16.2 导出格式

1. Markdown。
2. TXT。
3. DOCX。
4. PDF。

### 16.3 导出内容

完整记录导出应包含：

1. 标题。
2. 员工。
3. 部门。
4. 录音来源。
5. 创建时间。
6. 录音时长。
7. 总结。
8. 跟单。
9. 待办。
10. 待核对信息。
11. 逐字稿，可选。

导出时默认不包含：

1. 后端模型日志。
2. API Key。
3. 内部错误堆栈。
4. 无关审计日志。

## 17. 复用 ai-dayibin 的要求

ai-dayibin 已有录音总结能力，第一阶段应尽量复用，不要另造一套模型逻辑。

可复用内容：

1. 录音模板：会议纪要、业务复盘、客户跟进、红娘客户画像。
2. 逐字稿标准化。
3. 说话人别名。
4. 摘要卡片。
5. 思维导图 JSON。
6. Markdown 总结。
7. 导出格式。
8. 模型路由：`gpt-5.5` 优先，`kimi-k2.6` 兜底。
9. EasyAI/反代通道。

需要新增：

1. 招聘客户跟进模板。
2. 招聘五阶段跟进字段。
3. 招聘标签建议。
4. 招聘可复制 CRM 文本。
5. 部门权限和员工体系。
6. Chrome 扩展捕获网页录音。

5. 输出可复制跟进记录。
