# 大宜宾录音助手真实 Chrome 扩展侧栏 E2E 验收清单

更新时间：2026-06-17

用途：用于验证“真实侧栏卡死复盘与二次修复”在 Chrome 扩展侧栏里是否真正可用。自动化测试已经覆盖 DOM 回归；本清单补足 `chrome-extension://` 侧栏无法稳定由本地脚本直接接管时的人工验收和取证。

## 验收前准备

1. 后端服务保持可访问，登录页默认地址应指向当前可用后端。
2. Chrome 已加载本项目扩展，且扩展侧栏可通过扩展按钮打开。
3. Plaud 网页端已打开到一条录音详情页。
4. 准备一个 PNG/JPG/WebP 头像文件。
5. 准备一段可手动上传的录音文件。

## 必做流程

1. 打开 Plaud 网页端录音详情页。
2. 打开“大宜宾录音助手”Chrome 扩展侧栏。
3. 点击侧栏顶部头像或个人入口，进入“个人资料”。
4. 点击“一句话自我介绍”，确认光标出现。
5. 输入 `中文 abc123`，确认内容不丢失，页面不闪烁回退。
6. 依次点击旧密码、新密码、确认新密码，确认每个输入框都能聚焦和输入。
7. 选择头像文件，确认头像表单内显示文件名、大小、格式校验结果。
8. 点击“上传头像”，确认按钮显示上传中，完成后头像刷新；失败时头像表单内显示失败原因。
9. 切到“监听”页，点击识别当前页录音。
10. 出现候选录音后点击“生成纪要”，确认 300ms 内候选卡片显示“正在读取网页录音”或等价状态。
11. 如果 Plaud 直读失败，确认候选卡片内显示失败原因、“重新读取”、“手动上传/改用手动上传”、“复制诊断”。
12. 如果记录已创建但上传失败，确认候选卡片显示记录 ID，并提供“重试上传”。
13. 点击“手动上传”，确认进入上传页，标题自动带入候选标题，顶部显示“网页读取失败，建议上传从 Plaud 下载的原始录音文件”。
14. 选择录音文件，确认上传页显示文件名和大小。
15. 点击上传并生成纪要，确认进入详情页并显示处理状态条。
16. 依次点击首页、上传、历史、配置，确认导航仍可切换，无需去扩展管理页重新加载。
17. 在配置页点击“复制扩展诊断”，确认剪贴板 JSON 包含 `globalStatePhase`、`candidatesCount`、`lastScanError`、`serviceWorkerStartedAt`、`extensionVersion`。
18. 若出现“插件前端异常”面板，点击“复制诊断”，确认 JSON 包含当前 `view`、最近 `activeAction`、最近错误。

## 通过判定

只有以下条件全部满足，才算真实侧栏 E2E 验收通过：

1. 个人资料页输入框均可聚焦、可输入，输入内容不会因点击或后台录音状态更新丢失。
2. 头像选择、上传中、成功、失败都在头像表单内可见。
3. 点击“生成纪要”后候选卡片立即显示动作状态。
4. Plaud 直读失败时，候选卡片内有明确失败原因和手动上传入口。
5. 已创建记录但上传失败时，可以对同一记录重试上传。
6. 任何失败后，首页、上传、历史、配置仍可切换。
7. 前端异常和扩展后台诊断均可复制。
8. 普通交互不需要进入 `chrome://extensions` 重新加载扩展才能恢复。
